# LEDドットキャンバス プロジェクト憲法

## 1. 目的・ターゲット・利用シーン
- **目的**: 学園祭や展示会などのイベントで、来場者がスマートフォンを使ってドット絵を描き、それを会場 of 大型のLED表示器に反映させるためのシステム。
- **ターゲット**: 子どもから大人まで幅広い来場者、および会場のLED表示を制御するシステム管理者（裏方）。
- **利用シーン**: 
  - 来場者：会場のQRコードからスマートフォンでアクセスし、その場で描画・投稿する。
  - 管理者：PCやタブレットで管理画面を開き、投稿された作品の確認・LED表示の切替・スライドショー制御を行う。

## 2. 技術スタック
- **構造 (HTML)**: HTML5 (来場者用SPA、管理者用データテーブル構造)
- **装飾 (CSS)**: Vanilla CSS (来場者用はパステルポップ、管理者用は操作性を重視したクールなダッシュボードスタイル)
- **論理 (JS)**: Vanilla JavaScript + Node.js (Express) + SQLite3 (16x16グリッド描画、履歴管理、HTTP/CORS通信でのデータ同期、スライドショータイマー)

## 3. MVP（最小構成）の範囲
- **来場者用ページ (index.html, app.js)**:
  - 1つのHTMLファイル内でのSPA画面遷移。
  - 16x16キャンバス描画（タッチなぞり対応）、Undo/Redo、ニックネーム制限（30文字）。
  - 送信時に `POST http://localhost:3000/submissions` 経由でSQLiteデータベースへ保存。
- **管理者用ページ (admin.html, admin.css, admin.js)**:
  - **システム状態**: 接続状態表示、模擬接続テスト。
  - **スライドショー設定**: 自動巡回の開始/停止、表示時間（秒）、表示件数の設定。
  - **現在の表示情報**: LEDに表示されている作品のリアルタイムプレビュー。
  - **作品一覧**:
    - テーブル形式での表示（サムネイル、ID、ニックネーム、投稿日時、ピン留め、スライドショー対象、操作）。
    - フィルター（すべて/ピン留めのみ/スライドショー対象のみ/対象外のみ）およびニックネーム検索。
    - 操作アクション：拡大プレビュー（モーダル）、手動表示、削除（モーダル確認付き）。
  - **サーバー (server.js) と SQLite**:
    - `submissions` テーブルに `is_pinned`, `is_showing`, `is_new` を格納（初期カラム無ければ自動移行）。
    - データベースが空の場合に「猫」「ハート」「みかん」「星」「雲」の5件の初期ドット絵データを自動生成してSQLiteにインサート。
    - 各種操作（ピン、スライドショー対象、削除）をPATCH/DELETE APIでSQLiteに即座に反映。

## 4. デザイン・世界観
- **来場者用**: パステルポップ、ドット絵、手書き風デコレーション、白猫マスコット。
- **管理者用**: クール＆クリアなダッシュボード。ダークブルーを基調としたヘッダー、視認性の高いテーブル、操作しやすいトグルスイッチやボタン。
